# `data/` — local data artifacts (not tracked in git)

This directory holds working data for the CIE ingestion and research pipelines.
**Its contents are intentionally git-ignored** (see the `*.db`, `*.pkl`,
`data/models/`, `data/cie/*.pdf` rules in [`.gitignore`](../.gitignore)).
Published data flows through the Zenodo DOI, not through git blobs:
`scripts/ops/fetch_data.py` downloads the deposit and verifies SHA256
checksums, and large intermediates are regenerated by the pipeline scripts
listed below. (DVC was removed 2026-07-17 — the tooling had been dead
config since the migration off GCP. An earlier version of this note said
its remote bucket died with a deleted GCP project; the bucket in fact
survives, orphaned, in a live project — inventory and salvage map in
the maintainer-held recovered-buckets inventory.)

## Layout

| Path | Contents | Produced by |
|------|----------|-------------|
| `cie/databases/cie_rescued.db` | Curated SQLite of recovered + geocoded CIE Vol. I records (576 records passing automated extraction/geocoding validation, ~1.5k `text-embedding-004` vectors) | `scripts/data_pipeline/ingest_cie_rescued.py`, `scripts/attic/fix_rescued.py`, `scripts/data_pipeline/geocode_rescued_batch.py` |
| `cie/*.pdf` | Source CIE Vol. I scans (git-ignored) | manual download |
| `cie/pages/`, `cie/progress_*.json` | Per-page VLM extraction state | `scripts/data_pipeline/ingest_cie_all.py` |
| `models/` | Local ML model artifacts | training scripts |

## Provenance manifest

One entry per artifact in the layout table, plus the Zenodo deposit
(AGENTS.md rule 6: source, license, retrieval date, transform chain —
unknowns are recorded as unknown, not guessed). Recovered GCS artifacts
carry their provenance in the maintainer-held recovered-buckets
inventory (contact the maintainer for access).

### Zenodo deposit — `openetruscan_clean.csv`

- **What:** the published 6,567-row ML-ready corpus, version DOI
  [10.5281/zenodo.20075836](https://doi.org/10.5281/zenodo.20075836)
  (v1.0.0, published 2026-05-07).
- **Sources:** Larth Dataset (Vico & Spanakis, 2023; ~71% of rows) and CIE
  Vol. I extractions (~29%) — full citations in
  [`research/BIBLIOGRAPHY.md`](../research/BIBLIOGRAPHY.md).
- **License:** the deposit is CC-BY-4.0 (per the Zenodo record metadata).
  Upstream, as recorded in the `data_sources` table seeded by migration
  `c3f4d5e6a7b8`: Larth CC-BY-4.0; CIE Vol. I public domain (Pauli et al.,
  1893–).
- **Retrieval dates:** both upstream sources are recorded as retrieved
  2024-04-01 in `data_sources`. No more precise record survives.
- **Transform chain:** `seed_larth.py` (Larth CSV auto-download) →
  `normalize_inscriptions.py` → `merge_larth_metadata.py`, merged with the
  CIE ingestion chain below — see
  [`scripts/data_pipeline/README.md`](../scripts/data_pipeline/README.md).
- **Checksums:** SHA-256 pinned in
  [`scripts/ops/fetch_data.py`](../scripts/ops/fetch_data.py) and
  cross-listed in `research/v2/data/SHA256SUMS`; the Zenodo record carries
  its own MD5. Not repeated here.

### `cie/*.pdf` — CIE Vol. I source scans

- **Source:** studietruschi.org scans of the *Corpus Inscriptionum
  Etruscarum* (six Vol. I fascicles + the Vol. II indices). The exact
  source URLs are recorded in `cie/download_cie.sh`, recovered in the DVC
  snapshot (documented in the maintainer-held inventory), which also holds
  MD5s and byte sizes for all seven PDFs.
- **License:** the 1893– edition text is public domain; the terms under
  which studietruschi.org publishes its scans were never recorded —
  **unknown**, so the PDFs are not redistributed by this project.
- **Retrieved:** exact date unknown; copies existed by 2026-04-04 (the DVC
  snapshot date).
- **Transform chain:** none — these are the source artifacts.

### `cie/pages/`, `cie/progress_*.json` — extraction state

- **Source:** derived from `cie/*.pdf`.
- **License:** derivative of public-domain text.
- **Created:** regenerated on demand (git-ignored intermediates).
- **Transform chain:** `ingest_cie_all.py` (Gemini VLM page extraction;
  key from `.env`).

### `cie/databases/cie_rescued.db` — curated CIE records

- **Source:** derived from `cie/*.pdf`.
- **License:** derivative of public-domain text; geocodes and embeddings
  are project-generated.
- **Created:** original run date unknown (pre-squash); regenerable.
- **Transform chain:** the CIE ingestion chain in
  [`scripts/data_pipeline/README.md`](../scripts/data_pipeline/README.md)
  — VLM extraction → language/range/dedup surgery (now in
  `scripts/attic/`) → Gemini + Mapbox findspot geocoding →
  `fix_rescued.py` (four hand-mapped records) →
  `geocode_rescued_batch.py`. The ~1.5k vectors are Google
  `text-embedding-004`.

### `models/` — local ML checkpoints

- **Source:** trained by this project's training scripts on prod-corpus
  exports; base models `google/byt5-small` (Apache-2.0) and
  `xlm-roberta-base` (MIT).
- **License:** project-trained weights are MIT (repo license); base-model
  licenses as above.
- **Created:** original training dates unknown except where a checkpoint's
  own `training_metadata.json` records them.
- **Transform chain:** per checkpoint —
  The maintainer-held inventory documents every recovered
  checkpoint's training corpus, base model, and metadata. Expected local
  occupants: `byt5-v4`/`byt5-v5`
  ([`research/experiments/byt5_v4_vs_v5/`](../research/experiments/byt5_v4_vs_v5/README.md))
  and, once salvaged, `char-mlm-v1`/`lora-char-head-v1`
  ([`research/experiments/lacuna_restoration/`](../research/experiments/lacuna_restoration/README.md)).

Research-side data artifacts (label CSVs, frozen splits, anchors) carry
their provenance in [`research/data/README.md`](../research/data/README.md)
and [`research/v2/data/README.md`](../research/v2/data/README.md).


frozen splits → metrics → local API).

## Restoring `cie_rescued.db`

It is not in the working tree of fresh clones. **Regenerate** from the source
CIE Vol. I PDFs via the `scripts/data_pipeline/` chain
(`ingest_cie_rescued.py` → `geocode_rescued_batch.py`; the one-off repair
`scripts/attic/fix_rescued.py` documents four hand-mapped records from the
original run). This is the only self-contained path.
