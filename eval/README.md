# `eval/` — committed benchmark outputs

Each file in this directory is the JSON produced by a single run of a
**frozen** benchmark. The point of committing them is reproducibility:
the FINDINGS.md narrative cites specific numbers, and these are the
files those numbers come from.

## Naming convention

```
rosetta-eval-v1-<UTC-timestamp>.json
```

Generated by:

```bash
bash evals/rosetta_eval_v1.sh \
    --api-url https://api.openetruscan.com \
    --output auto
```

`--output auto` writes here with the correct timestamped name.

## What lives in each file

Top-level keys:

- `benchmark` — frozen-benchmark label (e.g. `rosetta-eval-v1`).
- `generated_at_utc` — ISO-8601 wall-clock time of the run.
- `api_url` — URL of the API under test (the "model" column).
- `random` — analytical random-retrieval baseline (no API traffic).
- `levenshtein` — edit-distance baseline against the Latin vocab.
- `model` — the model the API was serving when this ran.

See [`research/notes/reproduce-rosetta-eval-v1.md`](../research/notes/reproduce-rosetta-eval-v1.md)
for the full reproducibility manifest.

## When to add a new file

After a model change lands in prod that we expect to move the headline
numbers (e.g. v3 → v4 swap), re-run the benchmark and commit the
resulting JSON alongside the FINDINGS.md update.
